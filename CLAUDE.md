# LUCAS Agent Team — Оркестратор

## Про проєкт

**Сайт:** https://www.lucasua.com  
**Репозиторій:** https://github.com/lucasukraine/lucas-astro  
**Технології:** Astro.js (статичний сайт), GitHub Pages (custom domain: www.lucasua.com)  
**Мова контенту:** Українська

**Що робить LUCAS:**
- Аудит та МСФЗ-звітність
- SAF-T Connector (власна платформа для ДПС)
- Міграція з 1С/BAS на Odoo
- Трансфертне ціноутворення
- Due diligence, оцінка активів, юридичні послуги

**Цільова аудиторія:** CFO, фінансові директори, головні бухгалтери, власники бізнесу (B2B, Україна)

## Команда агентів

| Агент | Роль | Тригер |
|-------|------|--------|
| `planner` | Технічний планувальник | Перед будь-якою розробкою |
| `coder` | Astro.js розробник | Реалізація планів |
| `tester` | QA та перевірка | Після змін коду |
| `reviewer` | Code review | Перед публікацією |
| `seo-specialist` | SEO стратегія та технічне SEO | SEO-задачі |
| `content-maker` | Автор блогу | Написання статей |
| `fact-checker` | Перевірка фактів | Після кожної статті |

## Стандартні робочі процеси

### Нова функція сайту
```
planner → coder → tester → reviewer
```

### SEO-оптимізація сторінки
```
seo-specialist → coder → tester
```

### Нова стаття в блог
```
seo-specialist (ключові слова) → content-maker → fact-checker → reviewer
```

### Повний аудит сайту
```
seo-specialist + tester (паралельно) → planner → coder
```

## Правила оркестратора

1. Завжди починай з `planner` для технічних задач
2. `fact-checker` обов'язковий для будь-якого контенту про законодавство
3. Статті публікуються лише після approval від `fact-checker`
4. SEO-метадані перевіряє `seo-specialist` перед публікацією
5. Усі зміни коду проходять через `reviewer`
